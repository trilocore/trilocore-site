The problem

Smart contracts are still the soft target

Existing tools still miss the attacks that matter — computed jumps into orphan blocks, precision and rounding errors, multi-step kill chains, and business-logic flaws. Manual audits remain slow and expensive. Teams need fast, deep, automated coverage they can run themselves.

$3.4B*
Reported stolen across crypto in 2025. Smart-contract and protocol-logic bugs remain a primary attack surface.
$200M+*
Lost to individual smart-contract exploits in 2025 — including Cetus (~$224M) and Balancer (~$128M).
$150K+
A full professional manual audit runs $150K–$250K and still takes weeks — prohibitive for early-stage teams.

* Owner-supplied figures attributed to Chainalysis / Beosin 2025 reporting. Verify against the cited sources before relying on them.

What existing tools miss

  • P0
    Orphan block exploits — dead code with DELEGATECALL or SELFDESTRUCT reachable via computed JUMP at runtime. Static analyzers never walk these paths.
  • P1
    Precision & rounding errors — tiny per-call deltas drained across thousands of micro-swaps. Single-transaction analyzers never see the cumulative loss.
  • P1
    Composed attack chains — flash loan + oracle manipulation + reentrancy. Most tools analyze one transaction, missing the multi-step kill chain.
  • P1
    Gate bypass & access control — require(msg.sender == owner) bypassed via ungated writers to the owner slot. Attack-graph traversal is absent from existing tools.
  • P2
    Legacy & deprecated contracts — forgotten code that still custodies funds and gets re-exploited long after launch.
How it works

A five-phase pipeline, from bytecode to PoC

Paste a deployed address. Trilocore disassembles the bytecode, builds an attack graph, fuzzes at GPU speed, and confirms each finding by replaying it on a mainnet fork.

01 · RECON
Bytecode X-ray
Disassemble, build the CFG, extract selectors, resolve 4byte names, and detect compiler version and known CVEs.
02 · ANALYSIS
Attack graph
Bipartite selector↔slot graph. Finds ungated write paths, orphan blocks with dangerous ops, and access-control bypass routes.
03 · FUZZ
GPU execution
Parallel SIMD threads with multiple mutation strategies and live oracles for taint, reentrancy, integer, and control-flow bugs.
04 · VERIFY
CPU replay
Multi-layer verdict on an Anvil mainnet fork: revert check, PC reachability, ABI packing, fund-flow analysis, trace confirmation.
05 · AMPLIFY
Profit calculation
Flash-loan wrapping (Aave V3, Balancer, dYdX) and loop amplification. Net profit = extracted − gas − fees − MEV discount.
06 · REPORT
Foundry PoC
Auto-generated Foundry exploit test with real calldata, a profit assertion, and a plain-language narrative from Janus AI.
The platform

Detection and explanation, in one pipeline

BEVM Scanner catches what fuzzers and auditors miss. Janus AI explains what it finds in plain language — from bytecode X-ray to a working exploit PoC.

BEVM Scanner
bevm.trilocore.ai ↗
GPU-accelerated EVM bytecode fuzzer running parallel attack threads through a five-phase pipeline — recon, x-ray, attack-graph construction, fuzzing, and CPU replay confirmation on an Anvil mainnet fork. Generates working Foundry exploit PoCs automatically.
  • Visual workbench: Repeater, Scanner, Intruder, CFG graph, storage
  • 208 static bytecode detectors across 20 submodules
  • 13 runtime oracles: taint, reentrancy, integer, control-flow
  • Orphan block discovery via bipartite attack graph
  • Gate bypass: caller injection, wall cracker, Z3 solver
  • Mainnet-fork replay verdict on real on-chain state
Janus AI Model
janus.trilocore.ai ↗
A purpose-trained AI security model specialized in EVM vulnerability analysis. Janus translates raw scanner findings into plain-language exploit reports, generates attack narratives, and surfaces composable attack chains that numeric oracles alone would miss.
  • EVM-specialized — trained on real on-chain exploit data
  • Plain-language severity explanations for non-expert readers
  • Attack-chain composition — links findings into kill chains
  • Exploit narrative generation for audit reports
  • Contextual remediation recommendations
  • API-accessible — plug into any CI/CD security pipeline
